### Módulo 1: Introducción a Git

#### 1.1. ¿Qué es Git?
Git es un sistema de control de versiones distribuido, diseñado para gestionar proyectos de software de manera eficiente y colaborativa. Permite rastrear los cambios en el código fuente a lo largo del tiempo, facilitando el trabajo en equipo, la gestión de ramas, la integración de cambios y la reversión a versiones anteriores cuando sea necesario.

- **Definición y propósito**: Git es una herramienta que ayuda a los desarrolladores a llevar un registro de los cambios realizados en el código fuente de un proyecto, lo que facilita la colaboración y la gestión de versiones.
- **Breve historia y evolución**: Git fue creado por Linus Torvalds en 2005 como respuesta a las limitaciones de otros sistemas de control de versiones disponibles en ese momento. Desde entonces, ha experimentado un gran crecimiento y se ha convertido en el estándar de facto en el desarrollo de software.

#### 1.2. Ventajas de usar Git
Git ofrece numerosas ventajas en comparación con otros sistemas de control de versiones. Algunas de las principales ventajas incluyen:

- **Distribuido**: Cada desarrollador tiene una copia completa del repositorio, lo que permite el trabajo offline y una mayor flexibilidad en la colaboración.
- **Velocidad**: Git es extremadamente rápido en la gestión de cambios, la creación de ramas y la fusión de código.
- **Gestión de ramas eficiente**: Git facilita la creación y gestión de ramas, lo que permite a los equipos trabajar en paralelo en diferentes características o correcciones de errores.
- **Seguimiento preciso de cambios**: Git registra cada cambio realizado en el código, lo que facilita la identificación de quién hizo qué cambio y cuándo se realizó.
- **Integración con otras herramientas**: Git se integra fácilmente con una variedad de herramientas de desarrollo, como sistemas de seguimiento de problemas y sistemas de integración continua.
- **Comunidad activa y amplia adopción**: Git cuenta con una gran comunidad de usuarios y es ampliamente utilizado en la industria del desarrollo de software, lo que facilita encontrar recursos y soporte.

#### 1.3. Conceptos básicos
Para comprender Git, es importante familiarizarse con algunos conceptos fundamentales:

- **Repositorio**: Un repositorio Git es un directorio que contiene todos los archivos de un proyecto, así como la información de seguimiento de cambios de Git.
- **Commit**: Un commit es un registro de un cambio realizado en el repositorio. Cada commit tiene un mensaje descriptivo que indica los cambios realizados.
- **Branch**: Una rama en Git es una línea independiente de desarrollo que permite trabajar en nuevas características o correcciones de errores sin afectar la rama principal (generalmente llamada "master" o "main").
- **Merge**: La operación de merge combina los cambios de una rama en otra. Esto se utiliza para integrar el trabajo realizado en una rama de desarrollo de vuelta a la rama principal.
- **Remote**: Un repositorio remoto es una copia del repositorio en otro lugar, como un servidor en la nube. Permite la colaboración entre múltiples desarrolladores al proporcionar un lugar centralizado para compartir y sincronizar cambios.
